# 4K 高质量立体生成算法实现计划书

## 摘要

目标是在 `4k-stereo-synthesis-lab` 中实现一条独立的 4K stereo synthesis 实验链路，用于验证是否能在 4K 输入输出下超过当前 `Desktop2Stereo` depth-shift 路线和 `iw3 row_flow_v3_sym` 的视觉质量。

第一版不训练大模型、不直接修改 `Desktop2Stereo` 主项目。先实现可复现 benchmark、baseline、2-layer occlusion-aware synthesis，再决定是否接回主项目。

成功目标：

- RTX 2060 12GB：4K 可运行，Fast/轻量 Quality 模式可用。
- RTX 3090 / RTX 5070：Quality 4K 目标 60-90 FPS。
- HQ 4K：高端卡目标 30-60 FPS。
- 输出格式必须同时支持 `Half-SBS` 和 `Full-SBS`。
- 画质重点改善遮挡、边缘撕裂、空洞、前景漂浮、时序抖动。

## 实现阶段

### 阶段 1：Baseline 与评测框架

实现当前 `Desktop2Stereo` 类 depth-shift 基线，支持输入 `RGB + depth`，输出 `left_eye / right_eye / sbs / debug_info`。

新增模块：

- `src/stereo_runtime/baseline_shift.py`
- `src/stereo_runtime/io.py`
- `src/stereo_runtime/output.py`
- `src/stereo_runtime/metrics.py`
- `scripts/bench_4k.py`
- `scripts/compare_methods.py`

输出格式：

| 格式 | 输出尺寸 | 说明 |
|---|---|---|
| `half_sbs` | `W x H` | 左右眼各压缩到半宽后拼接，适合实时显示和多数 viewer |
| `full_sbs` | `2W x H` | 左右眼保持原始宽度后拼接，适合高质量保存和精细对比 |

验收标准：

- 单帧 4K RGB + depth 能输出 `Half-SBS`。
- 单帧 4K RGB + depth 能输出 `Full-SBS`。
- 能记录每阶段耗时、峰值显存、输出尺寸、FPS。
- baseline 输出与当前 `Desktop2Stereo` 主路径视觉接近。

### 阶段 2：Quality 4K 原型

实现 2-layer occlusion-aware stereo synthesis。

默认算法：

- 深度图分为 foreground/background 两层。
- 每层独立生成左右眼 warp。
- 左右眼生成保持对称约束。
- 使用 depth discontinuity + warp consistency 生成 occlusion mask。
- 只在 occlusion mask 区域做 edge-aware hole fill。
- 输出阶段统一通过 `output.py` 生成 `half_sbs` 或 `full_sbs`，算法内部始终保留左右眼原始分辨率。

新增模块：

- `src/stereo_runtime/layers.py`
- `src/stereo_runtime/occlusion.py`
- `src/stereo_runtime/hole_fill.py`
- `src/stereo_runtime/synthesis.py`

验收标准：

- 复杂边缘场景中，明显少于 baseline 的撕裂和空洞。
- RTX 3090 / RTX 5070 目标 4K 60-90 FPS。
- RTX 2060 12GB 可降级运行，目标 4K 30-60 FPS。
- `half_sbs` 与 `full_sbs` 输出在左右眼内容上保持一致，只改变拼接尺寸。

### 阶段 3：HQ 4K 原型

扩展为 3-4 layer synthesis，并加入更强的局部修复和时序稳定。

默认算法：

- `layers=3`，可切到 `2/4`。
- hole fill 仍只作用于 occlusion/high-gradient depth 区域。
- temporal cache 保存上一帧 depth/mask/warp，用于平滑遮挡边缘和减少抖动。
- 不做全帧大模型 refinement，只预留局部 refinement 接口。
- HQ 输出同样必须支持 `half_sbs` 和 `full_sbs`。

新增模块：

- `src/stereo_runtime/temporal.py`
- `src/stereo_runtime/refine.py`

验收标准：

- 比 Quality 4K 在遮挡复杂场景更稳定。
- RTX 3090 目标 4K 30-60 FPS。
- RTX 5070 可运行 HQ，但不强制 60 FPS。
- `full_sbs` 可用于离线画质评估，`half_sbs` 可用于实时 viewer 评估。

### 阶段 4：Desktop2Stereo 集成评估

实验仓库稳定后，输出接回 `Desktop2Stereo` 的最小集成方案。

建议接口：

```python
def synthesize_stereo(rgb, depth, config):
    """Return left_eye, right_eye, sbs, debug_info."""
```

建议配置：

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `backend` | `quality_4k` | `fast`, `quality_4k`, `hq_4k` |
| `layers` | `2` | Quality 默认 2，HQ 可 3-4 |
| `occlusion` | `true` | 启用遮挡 mask |
| `symmetric` | `true` | 强制左右眼对称生成 |
| `hole_fill` | `edge_aware` | `none`, `fast`, `edge_aware` |
| `temporal` | `true` | 启用时序稳定 |
| `output_format` | `half_sbs` | `half_sbs`, `full_sbs` |
| `debug_output` | `false` | 输出 mask/layer/warp 调试图 |

## 性能预算

| 模式 | 新增预算 | 目标硬件 | 目标 FPS |
|---|---:|---|---:|
| Fast 4K | `0-2 ms` | RTX 2060 12GB 起 | 60-90 |
| Quality 4K | `2-6 ms` | RTX 3090 / RTX 5070 | 60-90 |
| HQ 4K | `6-12 ms` | RTX 3090+ | 30-60 |
| Experimental | `12 ms+` | 高端卡实验 | 15-30 或离线 |

性能原则：

- 4K 主画面不做全帧大模型 refine。
- 深度推理保持 518 级别，再映射到 4K 合成。
- 算法内部保留左右眼原始分辨率，最后一步再决定 `Half-SBS` 或 `Full-SBS`。
- `Full-SBS` 输出尺寸为 `7680x2160`，显存和带宽压力高于 `Half-SBS`，优先用于画质评估和高端卡模式。
- 所有新增算法必须有降级开关。
- 先用 PyTorch CUDA tensor path 验证，再考虑 shader/TensorRT 化。

## 测试计划

必须覆盖：

- Correctness：左右眼尺寸、数值范围、`Half-SBS` 布局、`Full-SBS` 布局、batch/单帧输入。
- Benchmark：4K 单帧、多帧循环、FPS、显存峰值、每阶段耗时。
- 画质对比：baseline、iw3、Quality 4K 同场景输出。
- 输出一致性：同一组左右眼生成的 `Half-SBS` 和 `Full-SBS` 内容一致，仅缩放/拼接方式不同。
- 退化测试：关闭 occlusion、关闭 temporal、降低 layer 数、低显存模式。
- 时序测试：窗口拖动、游戏镜头移动、横向快速运动、文本 UI。

第一批测试场景：

- 桌面 UI 和浏览器文字。
- 游戏人物前景 + 复杂背景。
- 视频人物前景。
- 栏杆、头发、树枝等细边缘。
- 快速横向运动画面。

## 风险与默认决策

默认决策：

- 第一阶段只做几何和局部修复，不训练模型。
- 2-layer Quality 是主线，3-4 layer 是 HQ。
- RTX 2060 12GB 只要求可运行和轻量 Quality，不保证 HQ。
- RTX 3090 / RTX 5070 是主要 Quality/HQ 目标。
- 默认实时输出为 `Half-SBS`，`Full-SBS` 作为必备输出能力和高质量评估格式。
- 输出先以图片/张量/SBS 文件为主，稳定后再接 `Desktop2Stereo` viewer。

主要风险与降级：

| 风险 | 降级策略 |
|---|---|
| 4K 下新增耗时过高 | 降到 2-layer，关闭 HQ fill |
| RTX 2060 显存压力 | 只启用 Fast 或 2-layer lightweight |
| `Full-SBS` 带宽压力过高 | 实时默认 `Half-SBS`，`Full-SBS` 用于保存/评估 |
| hole fill 引入糊边 | fill 限制在 occlusion mask 区域 |
| temporal 造成拖影 | 提供 temporal strength 和关闭开关 |
| 与主项目 viewer 不匹配 | 先稳定离线/张量输出，再做集成 |
